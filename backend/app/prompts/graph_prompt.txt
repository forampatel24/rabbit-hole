NODE QUANTITY REQUIREMENTS:

Generate a DEEP, COMPREHENSIVE graph. The minimum is 25 nodes for ANY topic. For broader or more complex topics, scale up significantly:
- NARROW topic (e.g., a specific algorithm or technique): 25–34 nodes
- MEDIUM topic (e.g., a well-defined subfield): 35–49 nodes
- BROAD topic (e.g., a large domain like "Machine Learning"): 50–60+ nodes

These are minimums. Go beyond when the topic warrants it. Do not cap at 25 for broad topics.

DEPTH AND STRUCTURE REQUIREMENTS:

Build a graph with genuine depth (not just breadth). Structure it across these layers:

- PREREQUISITES (depth 0–1): Multiple foundational concepts required before starting. Break prerequisites into meaningful subtopics.
- CORE CONCEPTS (depth 1–2): The essential pillars of the topic. Each major pillar should be decomposed into 2–5 meaningful subtopics.
- INTERMEDIATE CONCEPTS (depth 2–3): Concepts that build on core ideas but are not yet advanced. These bridge the gap between foundational and specialized knowledge.
- ADVANCED CONCEPTS (depth 3–4): Deeper/specialized areas. Each advanced area should have its own subtopics.
- RELATED CONCEPTS (depth 1–3): Neighboring domains or complementary topics that connect to the main topic.
- TOOLS & FRAMEWORKS (depth 1–3): Where applicable, include practical tools, libraries, frameworks, and platforms.
- APPLICATIONS & USE CASES (depth 1–3): Real-world applications, industries, and practical implementations. Break applications into specific sub-applications.
- SUBTOPICS: Every major concept (core, intermediate, advanced) should be broken down into 2–5 meaningful subtopics wherever appropriate. Avoid monolithic nodes.

- At least 3–5 prerequisite concepts (broken into subtopics where meaningful)

Do NOT include:

- Filler nodes that are only tangentially related
- Trivial concepts that add no educational value
- Overly generic placeholders like "Other Applications" or "Miscellaneous"

EDGE RULES:

- Include prerequisite relationships from foundational to advanced

- Each node should have 2–5 edges on average (both incoming and outgoing)
- The root topic node should connect to its major branches
- Include at least one edge connecting related concepts from different branches
- Every node (except root and leaf nodes) should have both prerequisites and unlocks
